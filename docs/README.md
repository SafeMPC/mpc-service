# `/docs`

该目录汇总 MPC 基础设施相关的设计、计划与实施文档。推荐阅读顺序：

1. `mpc-development-plan.md` – **开发计划**：阶段拆解、任务列表、验收标准
2. `mpc-detailed-design.md` – **详细设计**：架构、流程图、协议与数据结构
3. `mpc-implementation-methodology.md` – **实施方法论**：开发流程、测试策略、CI/CD、风险管理
4. `mpc-implementation-strategy.md` – **总体实施策略**：多阶段路线、架构选型、演进路径
5. `server-initialization.md` – **部署/初始化指南**

补充说明：
- `mpc-development-plan.md` 中的任务状态需与代码实现保持同步
- 需要示例或模板时，可参考 https://github.com/golang-standards/project-layout/tree/master/docs

每次迭代完成后，请同步更新相关文档，确保设计与实现一致。
